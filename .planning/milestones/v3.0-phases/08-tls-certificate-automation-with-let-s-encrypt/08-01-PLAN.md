<task type="auto">
  <name>Task 2: Create cmd/cee-exporter/tls.go with all TLS builder functions</name>
  <files>cmd/cee-exporter/tls.go</files>
  <action>
Create a new file cmd/cee-exporter/tls.go in package main. This file will contain all TLS helper functions. The existing buildTLS() and logCertInfo() functions in main.go will be removed in Plan 02 (they are duplicated here temporarily during the parallel plan — Plan 02 will remove them from main.go).

Implement the following functions:

1. **buildManualTLS(certFile, keyFile string) (*tls.Config, error)**
   - Identical to the existing buildTLS() in main.go (tls.LoadX509KeyPair + MinVersion TLS 1.2)
   - This replaces buildTLS() — Plan 02 will rename the call site

2. **buildSelfSignedTLS(hosts []string) (*tls.Config, error)**
   - Use crypto/ecdsa, elliptic.P256(), crypto/rand, crypto/x509, crypto/x509/pkix, encoding/pem — stdlib only, no external deps
   - GenerateKey → MarshalECPrivateKey → x509.Certificate template with:
     - SerialNumber: big.NewInt(1)
     - Subject: pkix.Name{Organization: []string{"cee-exporter"}}
     - DNSNames: hosts (passed in; if empty, use []string{"localhost"} as fallback)
     - NotBefore: time.Now().Add(-time.Minute) — clock skew tolerance
     - NotAfter: time.Now().Add(365 *24* time.Hour) — 1 year validity
     - KeyUsage: x509.KeyUsageDigitalSignature
     - ExtKeyUsage: []x509.ExtKeyUsage{x509.ExtKeyUsageServerAuth}
   - CreateCertificate → pem.EncodeToMemory for both cert and key
   - tls.X509KeyPair(certPEM, keyPEM)
   - Return &tls.Config{Certificates: []tls.Certificate{cert}, MinVersion: tls.VersionTLS12}, nil

3. **buildAutocertTLS(domains []string, email, cacheDir string) (*autocert.Manager, *tls.Config, error)**
   - Import "golang.org/x/crypto/acme/autocert"
   - Validate: len(domains) == 0 → return error "acme_domains must not be empty"
   - If cacheDir == "" → cacheDir = "/var/cache/cee-exporter/acme"
   - Construct autocert.Manager:

     ```go
     m := &autocert.Manager{
         Prompt:     autocert.AcceptTOS,
         Email:      email,
         HostPolicy: autocert.HostWhitelist(domains...),
         Cache:      autocert.DirCache(cacheDir),
     }
     ```

   - Return m, m.TLSConfig(), nil
   - NOTE: Do NOT use acme.LetsEncryptURL field — let autocert use its default (production Let's Encrypt). Staging is an operator config concern, not code.

4. **startACMEChallengeListener(m *autocert.Manager, challengeAddr string) error**
   - If challengeAddr == "" → challengeAddr = ":443"
   - net.Listen("tcp", challengeAddr)
   - Start goroutine: srv := &http.Server{TLSConfig: m.TLSConfig()}; srv.ServeTLS(ln, "", "")
   - Log: slog.Info("acme_challenge_listener_started", "addr", challengeAddr)
   - Return nil (errors from ServeTLS logged inside goroutine, not fatal)

5. **logCertInfo(certFile string)**
   - Identical to existing logCertInfo() in main.go: slog.Info("tls_cert_loaded", "cert_file", certFile)
   - Plan 02 removes this from main.go

Required imports: crypto/ecdsa, crypto/elliptic, crypto/rand, crypto/tls, crypto/x509, crypto/x509/pkix, encoding/pem, fmt, math/big, net, net/http, time, log/slog, golang.org/x/crypto/acme/autocert.

IMPORTANT: This file must NOT have a build tag — it must compile on all platforms (linux, windows) with CGO_ENABLED=0. The autocert package is pure Go and has no C deps.
