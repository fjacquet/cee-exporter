<task type="auto">
  <name>Task 2: Implement pkg/prometheus/handler.go</name>
  <files>pkg/prometheus/handler.go</files>
  <action>
Create the file pkg/prometheus/handler.go with package name "ceeprometheus" (package ceeprometheus — avoids collision with the imported prometheus package).

Implement two exported functions:

1. NewMetricsHandler() http.Handler
   - Create a private registry: reg := prometheus.NewRegistry()
     DO NOT use prometheus.DefaultRegisterer — the default registry includes Go runtime metrics (GC, goroutines) irrelevant to this daemon.
   - Register 4 metrics via reg.MustRegister:
     a. prometheus.NewCounterFunc with Name="cee_events_received_total", Help="Total CEPA events received by the HTTP handler.",
        callback: func() float64 { return float64(metrics.M.EventsReceivedTotal.Load()) }
     b. prometheus.NewCounterFunc with Name="cee_events_dropped_total", Help="Total events dropped due to queue overflow.",
        callback: func() float64 { return float64(metrics.M.EventsDroppedTotal.Load()) }
     c. prometheus.NewCounterFunc with Name="cee_writer_errors_total", Help="Total errors returned by the event writer.",
        callback: func() float64 { return float64(metrics.M.WriterErrorsTotal.Load()) }
     d. prometheus.NewGaugeFunc with Name="cee_queue_depth", Help="Current number of events waiting in the async queue.",
        callback: func() float64 { return float64(metrics.M.QueueDepth()) }
   - Also register a bonus CounterFunc for cee_events_written_total (Name="cee_events_written_total",
     Help="Total events successfully written by the output writer.",
     callback: func() float64 { return float64(metrics.M.EventsWrittenTotal.Load()) })
     This is free (one extra line) and operationally useful for success-rate queries.
   - Return: promhttp.HandlerFor(reg, promhttp.HandlerOpts{})

   CRITICAL — callback safety: Each callback MUST only call .Load() or .QueueDepth() on pkg/metrics.M.
   No I/O, no mutex, no network calls inside callbacks. The callback runs in the scrape goroutine.

2. Serve(addr string) error
   - Create a dedicated http.ServeMux (NOT http.DefaultServeMux)
   - Register NewMetricsHandler() at "/metrics"
   - Build an http.Server{Addr: addr, Handler: mux} and call ListenAndServe()
   - Return the error (caller decides what to do with http.ErrServerClosed)

Imports needed:
  "net/http"
  "github.com/prometheus/client_golang/prometheus"
  "github.com/prometheus/client_golang/prometheus/promhttp"
  "github.com/fjacquet/cee-exporter/pkg/metrics"

No build tags needed — this file is pure Go, all platforms.
  </action>
  <verify>CGO_ENABLED=0 go build ./pkg/prometheus/...</verify>
  <done>pkg/prometheus/handler.go compiles cleanly with CGO_ENABLED=0; exports NewMetricsHandler and Serve; no DefaultRegisterer usage</done>
</task>
