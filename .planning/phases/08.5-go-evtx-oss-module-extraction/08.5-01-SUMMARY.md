# Phase 8.5 Plan 01: go-evtx OSS Module Extraction Summary

**One-liner:** Standalone github.com/fjacquet/go-evtx module with template-based BinXML EVTX writer, 12 passing tests, zero external dependencies, tagged v0.1.0 and indexed on pkg.go.dev.

## What Was Built

A new public GitHub repository `github.com/fjacquet/go-evtx` containing a standalone Go module for writing Windows Event Log (.evtx) binary files. The module:

- Exposes a simple three-method API: `New`, `WriteRecord`, `WriteRaw`, `Close`
- Encodes events as template-based BinXML (MS-EVEN6 format), compatible with python-evtx and Windows Event Viewer
- Uses only the Go standard library (zero external dependencies)
- Passes all tests on Linux/macOS without any Windows environment
- Is tagged v0.1.0 and indexed by the Go module proxy at proxy.golang.org

### API Adaptation (key design decision)

The `buildBinXML` function was adapted from `func buildBinXML(e WindowsEvent, offset uint32)` to `func buildBinXML(eventID int, fields map[string]string, offset uint32)`. This removes the dependency on `WindowsEvent` struct and makes go-evtx usable by any tool, not just cee-exporter.

Similarly, `collectSubstitutions(e WindowsEvent)` became `collectSubstitutionsFromFields(eventID int, fields map[string]string)`.
