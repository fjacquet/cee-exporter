    Create pkg/queue/queue_test.go with package declaration `package queue` (white-box — same package, needed to access q.ch directly for the drop-on-full test).

    Define a fakeWriter type at the top of the file (before any test functions):

    ```go
    type fakeWriter struct {
        mu     sync.Mutex
        events []evtx.WindowsEvent
        done   chan struct{}
    }

    func (f *fakeWriter) WriteEvent(_ context.Context, e evtx.WindowsEvent) error {
        f.mu.Lock()
        defer f.mu.Unlock()
        f.events = append(f.events, e)
        if f.done != nil {
            select {
            case f.done <- struct{}{}:
            default:
            }
        }
        return nil
    }

    func (f *fakeWriter) Close() error { return nil }
    ```

    Write TestEnqueue:
    - Reset metrics: metrics.M.EventsDroppedTotal.Store(0)
    - Create: fw := &fakeWriter{done: make(chan struct{}, 1)}
    - Create: q := New(10, 1, fw)
    - Start: q.Start(context.Background())
    - Defer: q.Stop()  (IMPORTANT: call Stop to drain before test ends)
    - Enqueue one event: e := evtx.WindowsEvent{EventID: 4663, CEPAEventType: "CEPP_FILE_WRITE"}
    - Call: ok := q.Enqueue(e)
    - Assert ok == true: if !ok { t.Fatal("enqueue returned false on non-full queue") }
    - Wait for worker: <-fw.done
    - Lock, check len(fw.events) == 1, unlock
    - Check events[0].EventID == 4663

    Write TestDropOnFull:
    - Reset metrics: metrics.M.EventsDroppedTotal.Store(0)
    - Create: fw := &fakeWriter{} (no done channel — we do NOT start the queue)
    - Create: q := New(2, 1, fw) — capacity 2
    - Do NOT call q.Start() — workers not running ensures channel fills without being drained
    - Fill the channel directly (white-box): q.ch <- evtx.WindowsEvent{EventID: 4663}; q.ch <- evtx.WindowsEvent{EventID: 4663}
    - Call: ok := q.Enqueue(evtx.WindowsEvent{EventID: 4663}) — must return false
    - Assert: if ok { t.Error("expected Enqueue to return false on full queue") }
    - Assert: if metrics.M.EventsDroppedTotal.Load() != 1 { t.Errorf(...) }
    - IMPORTANT: Since Stop() closes q.ch and expects workers to drain it, and we have a started worker that will be blocked, do NOT call q.Stop() after this test. Instead drain the channel manually:
      for len(q.ch) > 0 { <-q.ch }

    Write TestDrainOnStop:
    - Reset metrics: metrics.M.EventsDroppedTotal.Store(0)
    - Create: fw := &fakeWriter{done: make(chan struct{}, 3)}
    - Create: q := New(10, 2, fw) — 2 workers
    - Start: q.Start(context.Background())
    - Enqueue 3 events with different EventIDs (4663, 4660, 4670)
    - Call: q.Stop() — must block until all 3 events are processed
    - After Stop() returns: check len(fw.events) == 3 (all events written before Stop returned)
    - Note: No time.Sleep needed — Stop() guarantees drain before returning

    Do NOT use t.Parallel() — queue tests share metrics.M global state.
    Do NOT use time.Sleep for synchronization.
    Do NOT import testify or any external library — stdlib only.
