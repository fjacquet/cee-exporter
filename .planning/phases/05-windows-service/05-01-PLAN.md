<objective>
Add github.com/kardianos/service v1.2.4 as a direct dependency and refactor run() to accept a context.Context so the Windows SCM Stop() method can bridge into the daemon's shutdown path.

Purpose: The Windows SCM sends a stop control by calling Stop() — it does NOT generate POSIX signals. The current run() only exits on SIGTERM/SIGINT, so SCM stop would leave the service stuck in "Stopping" state until forcefully killed. Making run() ctx-aware lets the SCM wrapper cancel it cleanly. This change also benefits Linux (future sd_notify support).

Output: Updated go.mod/go.sum with kardianos/service, run(ctx context.Context) in main.go, updated service_notwindows.go shim, all tests passing on both platforms.
</objective>

Step 2 — Refactor run() in cmd/cee-exporter/main.go:
- Change signature from `func run()` to `func run(ctx context.Context)`.
- The function already creates its own internal `ctx, cancel := context.WithCancel(context.Background())` for the queue — rename it to `queueCtx, queueCancel` to avoid shadowing the parameter.
- The existing shutdown block currently blocks on `<-sig` (SIGTERM/SIGINT signal channel). Change the select to also handle `<-ctx.Done()` so the SCM wrapper can cancel the context to trigger shutdown:
  ```go
  select {
  case <-sig:
      slog.Info("shutdown_signal_received")
  case <-ctx.Done():
      slog.Info("shutdown_context_cancelled")
  }
  ```
- After the select, the rest of the shutdown logic (cancel queueCtx, httpServer.Shutdown, q.Stop) remains unchanged.
- Remove the internal `context.WithCancel(context.Background())` call for the outer ctx; only keep the queue-scoped context. The queue now uses `queueCtx` derived from the param ctx:
  ```go
  queueCtx, queueCancel := context.WithCancel(ctx)
  defer queueCancel()
  q.Start(queueCtx)
  ```
- After the select block, remove the now-redundant `cancel()` call (queueCancel is deferred). Keep the 30s shutdown timeout context.

NOTE: Do NOT use signal.Stop() after the select — the signal channel can remain active.

Step 3 — Update service_notwindows.go:
- The `runWithServiceManager` function currently has signature `func runWithServiceManager(runFn func())`. Update it to adapt to the new run signature:

  package main

  import "context"

  // runWithServiceManager runs the program directly on non-Windows platforms.
  // On Windows, this function is replaced by service_windows.go,

  func runWithServiceManager(runFn func(ctx context.Context)) {

  }
