**Section: `## TLS Setup` (DOC-03)**

Explain: TLS secures data in transit between cee-exporter and its clients. For most default PowerStore CEPA configurations, the CEPA publisher sends plain HTTP — verify with Dell support whether your firmware supports HTTPS for event publishing. TLS is recommended when cee-exporter is accessed over an untrusted network.

Sub-section `### Generate a Self-Signed Certificate`:

Include both methods:

Method 1 (openssl >= 1.1.1):
```bash
openssl req -x509 -newkey rsa:4096 -sha256 -days 3650 -nodes \
  -keyout server.key \
  -out server.crt \
  -subj "/CN=cee-exporter" \
  -addext "subjectAltName=IP:192.168.1.10,DNS:cee-exporter.corp.local"
```
Note: Replace the IP and DNS values with the actual address of your cee-exporter host. The `-addext subjectAltName` flag is required — Go TLS (>= 1.15) rejects certificates without Subject Alternative Names.

Method 2 (openssl < 1.1.1, using extfile):
```bash
cat > san.cnf <<'EOF'
[req]
distinguished_name = req_dn
x509_extensions = v3_req
prompt = no
[req_dn]
CN = cee-exporter
[v3_req]
subjectAltName = IP:192.168.1.10,DNS:cee-exporter.corp.local
EOF

openssl req -x509 -newkey rsa:4096 -sha256 -days 3650 -nodes \
  -keyout server.key -out server.crt -config san.cnf
```

Sub-section `### Configure cee-exporter for TLS`:
```toml
[listen]
addr      = "0.0.0.0:12228"
tls       = true
cert_file = "/etc/cee-exporter/tls/server.crt"
key_file  = "/etc/cee-exporter/tls/server.key"
```

Note: cee-exporter logs a WARN at startup if the certificate expires within 30 days.

**Section: `## Dell PowerStore CEPA Registration` (DOC-04)**

Explain: CEPA (Common Event Publishing Agent) is configured per-NAS-server in the PowerStore Web UI. Each NAS server must have its own Event Publishing Pool.

Sub-section `### Prerequisites`:
- Port 12228 (TCP) must be open from the PowerStore NAS server IP to the cee-exporter host
- Verify connectivity before configuring CEPA:
  ```bash
  nc -zv <cee-exporter-ip> 12228
  ```

Sub-section `### Create an Event Publishing Pool` — numbered steps:
1. Navigate to **Storage > NAS Servers** in the PowerStore Web UI
2. Select the target NAS Server
3. Go to the **Security & Events** tab > **Events Publishing** sub-tab
4. Click **Create new event publisher** (or **Modify** if one already exists)
5. Set a Publisher Name (e.g., `cee-exporter`)
6. Under **Publishing Pools**, click **Create Pool**
7. Set Pool Name (e.g., `default-pool`)
8. Add a CEPA Server:
   - Host: cee-exporter IP or FQDN
   - Port: `12228`
9. Under **Event Configuration > Post-Events** tab:
   - Click **Select all**
   - Uncheck: `CloseDir`, `OpenDir`, `FileRead`, `OpenFileReadOffline`, `OpenFileWriteOffline`
   - Leave Pre-Events and Post-Error-Events unchecked
10. Click **Apply**

Sub-section `### Protocol Notes`:
- cee-exporter responds to the RegisterRequest handshake with HTTP 200 OK and an **empty body** — this is required by the CEPA protocol. Any XML in the response causes a fatal parse error in the CEPA publisher.
- Heartbeat PUT requests are acknowledged within 3 seconds. cee-exporter processes events asynchronously via its internal queue.
- If using VCAPS mode (thousands of events per batch), use `gelf_protocol = "tcp"` to avoid UDP packet loss.

**Section: `## Troubleshooting`**

| Symptom | Cause | Fix |
|---------|-------|-----|
| `SDNAS_CEPP_ALL_SERVERS_UNREACHABLE` alert on PowerStore | Port 12228 unreachable | Verify firewall: `nc -zv <cee-exporter-ip> 12228` |
| TLS handshake fails: "certificate has no suitable IP SANs" | Certificate missing SAN | Regenerate cert with `-addext "subjectAltName=..."` |
| No events in Graylog despite CEPA configured | GELF UDP packet loss | Switch to `gelf_protocol = "tcp"` |
| RegisterRequest causes fatal CEPA error | Non-empty response body | Do not modify the RegisterRequest handler; cee-exporter returns empty 200 OK |
| Events arrive but wrong NAS server | CEPA configured on wrong NAS | Verify CEPA is configured on the NAS server that owns the target file system |

**Section: `## Building from Source`**


Requires Go 1.21+. No CGO dependencies (CGO_ENABLED=0).
