### Integration Gap

**MISS-01: Metrics server has no graceful shutdown path** (severity: low)

`pkg/prometheus/handler.go:Serve()` creates a local `*http.Server` that is never stored or returned. The daemon's shutdown sequence at `main.go:245` calls `httpServer.Shutdown()` for the CEPA listener only. On SIGTERM or SCM Stop(), the metrics HTTP server goroutine is leaked and the port is held until the OS reclaims it at process exit.

- **Affected requirement:** OBS-05 (implicit: configurable port should be properly released)
- **Affected flows:** Linux SIGTERM shutdown, Windows SCM Stop()
- **Workaround:** On clean exit the OS reclaims the port immediately; only a concern if the process needs to be restarted quickly on the same port before full exit
- **Fix:** Return `*http.Server` from `Serve()` (or accept context) and call `metricsServer.Shutdown(shutdownCtx)` in the shutdown sequence alongside the CEPA server shutdown
